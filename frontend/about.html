Understanding the Problem

A simple breakdown of TSP, Dynamic Programming, and Bitmasking.

What is the Travelling Salesman Problem?

The Travelling Salesman Problem (TSP) asks the following question: "Given a list of cities and the distances between each pair of cities, what is the shortest possible route that visits each city exactly once and returns to the origin city?"

It is an NP-hard problem in combinatorial optimization, important in theoretical computer science and operations research.

Dynamic Programming & Bitmasking

To solve TSP efficiently for small datasets, we use Dynamic Programming (DP) combined with Bitmasking.

1. Bitmasking

We represent the set of visited cities as an integer (mask). If the i-th bit is 1, City i is visited. For example, a mask of 5 (binary 0101) means City 0 and City 2 are visited.

2. The DP State

We define our DP state as dp[mask][pos], which represents the minimum cost to visit all cities in the 'mask' and ending at city 'pos'.

dp(mask, pos) = min( dist[pos][next] + dp(mask | (1 << next), next) )

Time Complexity

The brute force approach takes O(n!) time. Using DP with Bitmasking, we reduce this to O(n² 2ⁿ). This allows us to solve for up to ~20 cities in a reasonable time.

Space Complexity

The space complexity is O(n 2ⁿ) because we need to store the results of all subproblems in a 2D table of size (2ⁿ × n).

Real-world Applications

  • Logistics: Planning delivery routes for trucks to save fuel and time.
  • Manufacturing: Optimizing the path of a robotic arm during circuit board drilling.
  • Network Routing: Efficiently routing data packets through nodes in a network.
  • DNA Sequencing: Ordering DNA fragments to reconstruct the original strand.
